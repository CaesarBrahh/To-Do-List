## Project Overview

Build a terminal-based To-Do List application written entirely in Python and contained within a single file.

The program must:

- Display a titled, numbered to-do list.
- Allow the user to add items.
- All the user to remove items by number.
- Continue running until the user chooses to quit.
- Interact strictly through terminal input/output.

The application does not require persistant storage.

## Tech Stack (Strict requirements)

| Category | Technology | Version/Notes |
| Language | Python | 3.10+ recommended |
| Interface | Terminal (CLI) | stdin/stdout only |
| Files | Single File | All logic must exist in one .py file |
| Storage | In-Memory List | No file/database persistence |


## Setup Commands

No external dependencies required.

Run the program using: 
python todolist.py 

## Project Structure

Single file: todolist.py

## Core Features

1. Display List

Display title: "To-Do List"
Display items numbered starting at 1.
If empty, display just the title with no list items.

2. Main Menu Loop

After displaying the list, prompt user with:
- Add item
- Remove item
- Quit

The program must:
- Continue looping after add/remove
- Handle invlaid menu input gracefully
- Not crash on unexpected input

3. Add Item

Prompt: "Enter item: "
Append new item to list
Reject empty or whitespace-only input.
Re-display updated list after addition.

4. Remove Item

Prompt: "Enter item number to remove: "
Remove item by 1-based index.
Renumber list after removal.
Handle:
- Non-integer input
- Out-of-range numbers
- Attempt to remove from empty list

5. Quit

Exit cleanly when user selects Quit.

## Acceptance Criteria

AC1 - Initial Display

- Program starts by displaying:
	- "To-Do List" (title)
	- (Note that since the program has just started, we won't display any list items.)

AC2 - Add Item

- List is empty, then:
- User adds "Walk dog," then:
- List displays:
	- "To-Do List" (title)
	- "1. Walk dog"

AC3 - Remove Item

- Given list:
	- "To-Do List"
	- "1. A"
	- "2. B"
	- "3. C"
- User removes item 2, then list displays:
	- "To-Do List"
	- "1. A"
	- "2. C"

AC4 - Invalid Remove Input

- If
	- User enters non-integer
- Then:
	- Print error message
	- Do not modify list
	- Continue loop / Re-prompt

- If
	- User enters number outside range
- Then:
	- Print error message
	- Do not modify list
	- Continue loop / Re-prompt

AC5 - Invalid Menu Input

- If
	- User enters invalid menu choice
- Then
	- Print Invalid choice.
	- Continue loop / Re-prompt

AC6 - Quit

- If
	- User selects Quit
- Then
	- Program terminates window without error
